# 课堂答题工具 PRD

## 一、产品概述

课堂答题工具是一个基于本地题库文件的轻量级答题网站。

老师可以提前准备好题库文件，网站会自动解析题目并生成答题流程。学生或老师可以逐题作答，系统提供即时反馈，并在最后生成正确率统计。

本产品重点在于：

- 简洁清晰的答题体验
- 单题模式减少干扰
- 即时正反馈增强课堂互动
- 快速部署和使用

第一版不包含后台系统和数据库。

---

## 二、产品目标

解决的问题：

课堂上展示选择题通常依赖：

- PPT
- 纸质练习
- 口头提问

这些方式：

- 没有互动
- 没有即时统计
- 学生参与感弱

本产品希望提供：

一个类似测评网站体验的课堂答题系统。

特点：

- 单题逐步答题
- 即时判定对错
- 右上角显示统计信息
- 最终展示正确率结果

---

## 三、用户角色

### 老师

职责：

- 准备题库文件
- 选择题集
- 在课堂展示题目

### 学生

职责：

- 阅读题目
- 选择答案
- 查看即时反馈

---

## 四、产品结构

第一版包含四个页面：

1. 首页
2. 答题页
3. 结果页
4. 后台入口（占位）

---

## 五、页面说明

### 1 首页

功能：

展示题集列表。

页面内容：

- 网站标题
- 题集列表
- 开始答题按钮
- 进入后台按钮（未来功能）

示例：

语文练习  
历史选择题  
DSE 常见字  

点击题集进入答题。

---

### 2 答题页

核心页面。

页面结构：

右上角统计：

- 当前题号
- 总题数
- 正确数量
- 错误数量

示例：

第 3 / 10 题  
正确：2  
错误：1  

中间：

题干

下方：

选项按钮。

示例：

A  
B  
C  
D  

---

### 答题逻辑

用户点击选项后：

系统立即判断对错。

答对：

- 选项变为绿色
- 显示勾号
- 触发轻量彩带动画

答错：

- 选项变为红色
- 显示正确答案

1 秒后自动进入下一题。

---

### 3 结果页

答题完成后展示统计信息。

显示内容：

- 总题数
- 正确题数
- 错误题数
- 正确率

示例：

共 10 题  
正确 7  
错误 3  
正确率 70%

按钮：

- 再做一次
- 返回首页

---

### 4 后台入口（占位）

首页包含按钮：

进入后台

点击后进入一个占位页面。

内容：

未来将支持：

- 上传题库
- 管理题集
- 查看课堂统计

第一版不实现后台功能。

---

## 六、题库来源

第一版所有题目来自本地题库文件。

题库文件放置在：

```

/data

```

示例文件：

```

quizSets.ts

```

---

## 七、题库数据结构

### QuizSet

一个题集。

字段：

- id
- title
- description
- questions

示例：

```

{
id: "chinese-basic",
title: "简繁字练习",
description: "DSE 常见简繁字",
questions: [...]
}

```

---

### Question

每道题结构。

字段：

- id
- stem
- options
- correctIndex
- explanation（可选）

示例：

```

{
id: "1",
stem: "“说”的繁体字是？",
options: ["说","語","說","話"],
correctIndex: 2,
explanation: "“说”的繁体字是“說”。"
}

```

---

## 八、交互原则

### 单题模式

页面一次只展示一题。

减少视觉干扰。

---

### 即时反馈

点击选项后立即显示结果。

增强参与感。

---

### 自动切题

答题完成后：

1 秒自动进入下一题。

---

### 固定统计

右上角统计信息始终可见。

包含：

- 当前题号
- 总题数
- 正确数
- 错误数

---

## 九、第一版功能范围

### 必须实现

- 读取本地题库
- 题集选择
- 单题答题
- 即时反馈
- 统计显示
- 结果页

---

### 不包含

- 数据库
- 登录系统
- 老师录题系统
- Excel 导入
- 班级系统
- 成绩保存

---

## 十、视觉方向

参考：

测评类网站答题界面。

视觉特点：

- 页面留白
- 大字号题干
- 大按钮选项
- 简洁配色

- 勾号动画
